# 深色模式浏览器测试指南

## 📋 测试前准备

### 1. 清除浏览器缓存的方法

#### Chrome/Edge
1. 按 `Ctrl + Shift + Delete` (Windows) 或 `Cmd + Shift + Delete` (Mac)
2. 选择"缓存的图片和文件"
3. 时间范围选"全部时间"
4. 点击"清除数据"

**或者**使用开发者工具:
1. 按 `F12` 打开开发者工具
2. 右键点击浏览器的刷新按钮
3. 选择"清空缓存并硬性重新加载"

**或者**使用无痕模式测试:
- 按 `Ctrl + Shift + N` (Chrome)
- 按 `Ctrl + Shift + P` (Firefox)
- 按 `Ctrl + Shift + T` (Edge)

#### Firefox
1. 按 `Ctrl + Shift + Delete`
2. 勾选"缓存"
3. 点击"立即清除"

#### Safari
1. 按 `Cmd + Option + E`
2. 或在菜单中选择"开发" > "清空缓存"

### 2. 访问应用

打开浏览器,访问: **http://localhost:3000**

## 🧪 测试步骤

### 测试 1: 检查主题切换按钮

**预期结果**:
- ✅ 左下角有一个固定的圆形按钮
- ✅ 按钮有阴影和边框
- ✅ 里面有一个图标 (月亮🌙 或 太阳☀️)

### 测试 2: 点击按钮切换到深色模式

**操作**: 点击左下角的按钮

**预期结果** (应该立即发生):
1. **背景颜色变化**:
   - 从: 浅灰到白色的渐变 (`from-slate-50 to-slate-100`)
   - 变: 深灰到黑色的渐变 (`from-slate-900 to-slate-800`)

2. **文字颜色变化**:
   - 标题: 从黑色 → 白色
   - 副标题: 从深灰 → 浅灰
   - 描述文字: 从深灰 → 浅灰

3. **卡片颜色变化**:
   - 从: 白色背景 + 浅灰边框
   - 变: 深灰背景 (`bg-slate-800`) + 深色边框

4. **按钮图标变化**:
   - 从: 🌙 月亮图标
   - 变: ☀️ 太阳图标 (金黄色)

### 测试 3: 切换回浅色模式

**操作**: 再次点击左下角按钮

**预期结果**:
- 所有颜色恢复到测试 2 之前的状态
- 太阳图标变回月亮图标

### 测试 4: 测试页面持久化

**操作**:
1. 切换到深色模式
2. 按 `F5` 或点击刷新按钮
3. 或关闭标签页重新打开

**预期结果**:
- ✅ 刷新后页面仍然保持深色模式
- ✅ 不需要重新点击按钮

### 测试 5: 测试所有页面

**操作**: 在不同页面之间切换

**测试页面**:
1. `/` - 欢迎页
2. `/info` - 信息采集页
3. `/questionnaire` - 问卷页
4. `/results` - 结果页 (需要先完成测试)

**预期结果**:
- ✅ 所有页面都正确响应主题切换
- ✅ 所有页面的样式都一致
- ✅ 每个页面都有左下角的主题切换按钮

### 测试 6: 测试自动主题检测

**操作**:
1. 在系统设置中修改主题 (Windows: 设置 > 个性化 > 颜色)
2. 选择"浅色" → 重启浏览器 → 访问应用
3. 选择"深色" → 重启浏览器 → 访问应用

**预期结果**:
- 系统浅色 → 应用浅色
- 系统深色 → 应用深色

## 🔍 诊断检查清单

如果深色模式不工作,请检查以下项:

### 在浏览器中检查 HTML 元素

1. 按 `F12` 打开开发者工具
2. 切换到 "Elements" 或"元素"标签
3. 查看 `<html>` 元素

**应该看到**:
```html
<!-- 深色模式时 -->
<html lang="zh-CN" class="dark">

<!-- 浅色模式时 -->
<html lang="zh-CN" class="">
```

**如果 `class` 没有变化**:
- 说明 next-themes 没有正常工作
- 检查浏览器控制台是否有错误

### 检查 CSS 变量

在开发者工具的 Console 中执行:

```javascript
// 查看当前的背景颜色变量
getComputedStyle(document.documentElement)
  .getPropertyValue('--background')

// 深色模式应该返回: #0a0a0a
// 浅色模式应该返回: #ffffff
```

### 检查 localStorage

在开发者工具的 Console 中执行:

```javascript
// 查看保存的主题设置
localStorage.getItem('theme')

// 应该返回: "light", "dark", 或 "system"
```

### 检查是否有 JavaScript 错误

1. 打开开发者工具 (F12)
2. 切换到 "Console" 或"控制台"标签
3. 查看是否有红色错误信息

**常见错误**:
- `useTheme must be used within a ThemeProvider`
- `next-themes` 相关错误
- `localStorage` 相关错误
